# セキュリティポリシー

## サポート対象バージョン

| バージョン | サポート状況 |
|---------|-----------|
| 最新リリース | はい |
| 古いバージョン | いいえ |

## 脆弱性の報告

**セキュリティの脆弱性についてpublic issueを作成しないでください。**

代わりに、GitHub Security Advisoriesを使用して非公開で脆弱性を報告してください：

[脆弱性を報告する](https://github.com/withkynam/vibecode-pro-max-kit/security/advisories/new)

> **注意：** 「脆弱性を報告する」ボタンを表示するには、リポジトリ設定でPrivate Vulnerability Reportingが有効になっている必要があります。

### 対応タイムライン

- 報告受領から**48時間以内**に確認
- **7日以内**に初期評価と重大度分類
- プロセス全体を通じて進捗をお知らせします

## スコープ

### 対象範囲

- **フックスクリプト**（`privacy-block.cjs`、`scout-block.cjs`）——エージェントがブロックされたリソースにアクセスできるバイパス脆弱性
- **install.sh** ——サプライチェーン攻撃、コマンドインジェクション、パストラバーサル
- **エージェントプロンプト** ——フェーズロックやツール制限をバイパスするプロンプトインジェクション
- **プライバシーリーク** ——ガードレールがあるにもかかわらず、エージェントがクレデンシャル、APIキー、機密ファイルにアクセスする問題
- **System-reminder / subagent-initインジェクション** ——エージェントの動作を操作するコンテキストインジェクション攻撃

### 対象外

- Claude CodeまたはCodex自体の脆弱性（それぞれAnthropicまたはOpenAIに報告してください）
- ユーザーのプロジェクトコードの問題（ハーネスではないもの）
- メンテナーに対するソーシャルエンジニアリング攻撃

## 開示タイムライン

- 確認された脆弱性は**30日以内**の修正を目指します
- 修正リリース後に調整された開示を行います
- リリースノートで報告者をクレジットします（匿名希望の場合を除く）

## セーフハーバー

善意で行われたセキュリティリサーチは認可されたものとみなします。善意で行動し、このポリシーに従う研究者に対して法的措置を取ることはありません。

リサーチが対象範囲に該当するか不明な場合は、テストの前にご連絡ください——スコープの明確化を喜んでお手伝いします。
